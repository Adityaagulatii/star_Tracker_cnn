# Star Tracker CNN

Comparing three U-Net variants for star detection and centroid localisation on real DSS2 sky images.

## Models

| Model | Architecture | Params |
|---|---|---|
| **UNet** | Double Conv blocks (Conv→ReLU×2) with skip connections | 481,745 |
| **MobileUNet** | Inverted Residual blocks (expand→depthwise→project) | 254,969 |
| **ELUNet** | Single Conv blocks (Conv→BN→ReLU×1), base channels = 8 | 59,625 |

All models take a single-channel 256×256 sky image and output a binary segmentation map marking star locations.

## Results

| Model | F1 Score | Centroid Error | Inference Time |
|---|---|---|---|
| UNet | 0.928 | 0.39 px | 5.54 ms |
| MobileUNet | 0.884 | 0.49 px | 17.81 ms |
| ELUNet | 0.910 | 0.45 px | 8.13 ms |

## Dataset

250 real sky images downloaded from the **DSS2 Red Survey** via NASA SkyView (200 train / 50 val).
Ground truth segmentation maps are generated by brightness thresholding and cross-referenced with the **Gaia DR3** star catalog.

## Files

| File | Description |
|---|---|
| `prepare_data.py` | Downloads DSS2 sky images and generates segmentation map labels |
| `dataset.py` | PyTorch Dataset class — loads `.npy` files for training |
| `model.py` | UNet, MobileUNet, and ELUNet architecture definitions |
| `loss.py` | Binary cross-entropy segmentation loss |
| `train.py` | Training loop — saves best checkpoint per model |
| `evaluate.py` | Evaluates all three models — F1, centroid error, inference time, visual outputs |

## Usage

**Step 1 — Install dependencies**
```
pip install torch torchvision numpy matplotlib scipy tqdm astropy astroquery
```

**Step 2 — Download dataset**
```
python prepare_data.py
```

**Step 3 — Train models**
```
python train.py --model unet
python train.py --model mobileunet
python train.py --model elunet
```

**Step 4 — Evaluate**
```
python evaluate.py
```
